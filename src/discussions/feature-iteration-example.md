# 示例：功能迭代讨论记录

## 议题

是否要在 v1 引入「XX 功能」？如果要，引入的边界与节奏如何？

## 结论（先写结论，后写过程）

- v1 **只做最小可用能力**：满足核心用例即可
- 复杂能力（权限细分/多租户/插件化）延后到 v2/v3
- 需要补一篇设计文档到 `design/`，并在评审通过后沉淀 ADR

## 讨论要点

- 约束：当前人力有限，且需要兼容旧数据
- 风险：一次性做大容易拖慢主线迭代
- 可行路径：先提供稳定接口，后续以可选字段/新端点演进

## 待办

- [ ] 在 `design/` 输出方案对比与接口草案
- [ ] 评审通过后写 ADR，明确最终边界与版本策略
