# ADR-0001：示例

## 状态

已接受（Accepted）

## 背景

我们需要一个“可追溯”的方式，记录架构与工程上的关键决定，避免以后反复争论“为什么当时这么做”。

## 决策

采用 **ADR** 作为决策沉淀形式：

- 重要决策必须有一篇 ADR
- 讨论过程放 `discussions/`，最终结论链接到对应 ADR

## 备选方案

- 只在 Issue/PR 里讨论：信息分散，长期难检索
- 只写在 README：容易膨胀、缺乏结构化

## 影响与后续

- 后续新增 ADR 时，从 `0002` 开始编号
- 评审通过后再合并，确保“已接受”的 ADR 足够稳定
